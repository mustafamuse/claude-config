# claude-config

My personal [Claude Code](https://claude.com/claude-code) setup — agents, hooks, skills, output styles, and global settings. Curated against guidance from Anthropic engineering, Boris Cherny, Cat Wu, Thariq Shihipar, and the Cursor team (Eric Zakariasson) as of May 2026.

> The full design rationale lives at `docs/claude-code-setup.md` inside [mustafamuse/irshad-center](https://github.com/mustafamuse/irshad-center).

---

## What's in here

| Path | Purpose |
|---|---|
| `CLAUDE.md` | Global rules loaded in every project (code style, verification primacy, anti-rot protocol) |
| `settings.json` | Env vars, hook wiring, `defaultMode: plan`, statusline, plugin enable list |
| `agents/` | `verify-app`, `code-explorer`, `security-reviewer`, `migration-reviewer`, `worktree-worker`, `ReadOnly`, `staff-reviewer` |
| `hooks/` | Pre/PostToolUse, UserPromptSubmit, Stop, SessionEnd, WorktreeCreate handlers |
| `skills/` | Workflow skills (`/notes`, `/babysit`, `/autopr`, `/swarm`, `/feature-gan`, `/context-budget`, etc.) |
| `output-styles/` | `review-mode.md` — terse, evidence-first output style |
| `commands/` | `/create-pr` slash command |
| `zsh/claude-aliases.zsh` | Worktree aliases (`za`/`zb`/`zc` + `zl`/`zd`) |
| `install.sh` | Idempotent installer — symlinks repo paths into `~/.claude/` |

## Install on a fresh machine

```bash
git clone git@github.com:mustafamuse/claude-config.git ~/dev/claude-config
cd ~/dev/claude-config
./install.sh
```

Then in `~/.zshrc`:

```bash
[ -f ~/dev/claude-config/zsh/claude-aliases.zsh ] && source ~/dev/claude-config/zsh/claude-aliases.zsh
```

```bash
source ~/.zshrc
```

Restart Claude Code to pick up `settings.json`.

The installer backs up any pre-existing files to `~/.claude/<path>.preinstall-<timestamp>` before symlinking. It's idempotent — re-running detects correct symlinks and skips.

## Machine-specific notes

- `settings.json` references `/opt/homebrew/bin/bun` in the `claude-hud` statusline command. On Linux or non-Homebrew Mac, change this to `$(command -v bun)` output.
- The `claude-hud` statusline plugin is installed via `claude plugin install claude-hud@claude-hud --scope user` — the marketplace is registered in `settings.json`.
- Hooks shell out to `gh`, `git`, `jq`, and `bun` — all expected on `$PATH`.

## Per-machine state stays out

These directories under `~/.claude/` are **deliberately not in this repo** — they contain conversation history, memory files with business context, and runtime caches:

- `projects/` — conversation logs + memory files
- `plugins/` — cached marketplace plugins (regenerated by `claude plugin install`)
- `sessions/`, `backups/`, `cache/`, `paste-cache/`, `file-history/`, `plans/`, `debug/`, `ide/`, `statsig/`, `telemetry/`, `todos/`, `tasks/`, `session-env/`, `shell-snapshots/`
- `history.jsonl`, `settings.local.json`, `mcp-needs-auth-cache.json`, `*.bak*`

## Design principles

1. **Verification is the #1 quality lever** — per Boris Cherny. Hence the `verify-app` agent and adversarial prompts in `CLAUDE.md`.
2. **Three layers of attention** — always-on (`CLAUDE.md`, hooks), path-attached (skills with `paths:` frontmatter), and slash-invoked (workflow skills, agents). Most of it is dormant until the right trigger.
3. **Leaf-node restriction** (Erik Schluntz) — high-risk paths in projects' `CLAUDE.md` files route through specialty agents (`security-reviewer`, `migration-reviewer`).
4. **Context-rot mitigation** — `/notes` skill + NOTES.md pattern + 400k auto-compact window + UserPromptSubmit git-context injection.
5. **Cost-aware** — the `Stop` hook is a cheap bash gate, not an agent spawn. The `MAX_THINKING_TOKENS` cap and forked subagent context save money on cheap tasks.
